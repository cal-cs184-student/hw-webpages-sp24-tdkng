Task 2: Antialiasing by Supersampling

In resolve_to_framebuffer(), we made a few significant changes. Using sample_buffer, our algorithm supersamples the rgb values at points by using a sample_buffer with a larger size (and higher resolution) than that of rgb_framebuffer_target. We use a nested for-loop to iterate through all the samples in a pixel and take the average r, g, and b values. This color gets outputted to the frame buffer and colors the entire pixel. Overall, our modification mainly only changed the way that the internal (sample) buffer was translated to the screen (frame) buffer.

In the other functions, we mainly just fixed the sample buffer to reflect the new changes. Specifically for each function:

fill_pixel(): we fixed the index to reflect the changes of the increased size in the sample buffer.

rasterize_triangle(): we changed the start indices and step sizes for iterating over the bounding boxes since we have to iterate at a higher resolution to implement supersampling. We also had to fix the way we call fill_pixel so that it would fill the correct pixel on the larger sample buffer for points and lines, which are not supersampled.

set_sample_rate(): we fixed the size of the sample buffer to scale based on the inputted rate.

set_framebuffer_target(): we increased the size of the sample buffer by a factor of the sample rate.

Supersampling is useful because it allows us to get a more accurate color for pixels that reduces jaggedness and aliasing overall. In effect, the images seem to look sharper, clearer, and better in quality. We used supersampling to antialias our triangles by taking multiple samples for a single pixel thus allowing us to get a more accurate rgb value for each pixel that would decrease aliasing.

Task 4: Barycentric Coordinates

Barycentric coordinates are a type of coordinate system based on proportions between the vertices of a triangle and their corresponding opposite edges. Each vertex of a triangle represents a distinct value or coordinate in some space. The proportion between the two line segments formed by the line going through a set of 2-D Euclidean coordinates and a vertex defines the Barycentric coordinate corresponding to that vertex. These coordinates define the proportional distance from the point to each vertex and sum to 1. An application of this system is to perform linear interpolation between the vertices of a triangle, which can be used for texture and color mapping such as in the following example.

The top left, top right and bottom vertices of the above triangle are set to 100% red, green and blue respectively. Every point within the triangle is assigned a color by linear interpolation via a weighted sum of the distances from the point to each vertex.

Task 5: Pixel Sampling

Pixel sampling is a method of populating the screen space frame buffer with the material of a texture map. Given a set of coordinates in texture space, we can get the corresponding coordinates in screen space and vice versa using Barycentric coordinates. This allows us to map the texture to our triangle mesh. We implemented texture mapping by picking texture coordinates based on the type of sampling being used. In code, we had to calculate the barycentric coordinates of each point in order to compute the coordinates necessary for retrieving the appropriate texel.

Nearest-pixel sampling maps the texture space value nearest to the point corresponding to the screen space point. Bilinear interpolation interpolates a texture space value between the four defined points closest to the point corresponding screen space point.

Task 6: Level Sampling

Level sampling, or mipmapping, is a method of texture minification that involves storing multiple “levels” of a texture image at successively lower resolution and lower maximum signal frequency. These “copies” of the original texture file are called mipmaps. When sampling a point in texture space, we sample from the file whose resolution approximates the screen sampling rate. This allows us to map a texture while minimizing aliasing in higher frequency areas and minimizing blurring in lower frequency areas.

We implemented level sampling by computing the differentials of the coordinates in texture space with respect to those of the screen space and computing the appropriate level using the equation derived in lecture. We use this level to find the corresponding mipmap texture file that is sampled when mapping to the screen space.

Pixel sampling uses minimal additional memory but suffers a notable performance degradation due to the additional lerp and sample operations. Its antialiasing power is less than supersampling but still good.

Level sampling is very speed efficient but takes up more storage due to saving multiple “copies” of the texture image file. It is very powerful for antialiasing because it can dynamically select the level of minification for different areas.

Supersampling uses notable additional memory from saving the colors of subpixels and suffers a major performance hit, especially for larger sample rates. However, it has great antialiasing power and can be scaled.
